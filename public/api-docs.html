CoreLink Helpdesk API

Integrate support ticket functionality into your applications.

Base URL

https://corelink.dev/api/helpdesk/v1

Authentication

All API requests require an API key passed in the X-API-Key header.

X-API-Key: your-api-key-here

API keys are project-scoped. Contact CoreLink to obtain an API key for your project.

If your API key has explicit permissions assigned, the usage endpoints require usage.read.

Endpoints

POST /tickets

Create a new support ticket. Supports file attachments via multipart/form-data.

Request Body

Field Type Required Description
title string Yes Ticket title (max 255 chars)
content string Yes Detailed description of the issue
submitter_name string Yes Name of the person submitting
submitter_email string Yes Email of the person submitting
submitter_user_id string No Your application's user ID (for reference)
priority string No low, medium, high, critical
type string No bug, feature, question, task
metadata object No Any additional structured data (JSON)
attachments[] file[] No Up to 10 files (images, PDF, Word, Excel, TXT, CSV). Max 10MB each.

Example Request (JSON)

{
  "title": "Unable to export inventory report",
  "content": "When I click the Export button on the inventory page, nothing happens.",
  "submitter_name": "John Smith",
  "submitter_email": "john@example.com",
  "submitter_user_id": "user_12345",
  "priority": "high",
  "type": "bug",
  "metadata": {
    "browser": "Chrome 120",
    "page_url": "/inventory/reports"
  }
}

Example Request with Attachments (multipart/form-data)

curl -X POST https://corelink.dev/api/helpdesk/v1/tickets \
  -H "X-API-Key: your-api-key" \
  -F "title=Bug with screenshot" \
  -F "content=See attached screenshot for the error." \
  -F "submitter_name=John Smith" \
  -F "submitter_email=john@example.com" \
  -F "priority=high" \
  -F "attachments[]=@screenshot.png" \
  -F "attachments[]=@error-log.txt"

Response (with attachments)

{
  "data": { ... ticket data ... },
  "attachments": [
    {
      "id": 1,
      "filename": "screenshot.png",
      "mime_type": "image/png",
      "size": 45678,
            "is_image": true,
            "url": "https://corelink.dev/api/helpdesk/v1/attachments/1/download",
            "view_url": "https://corelink.dev/api/helpdesk/v1/attachments/1/view"
    }
  ],
  "message": "Ticket created successfully"
}

Attachment URLs: url is the download endpoint for any attachment. view_url is only included for image attachments and serves the file inline for preview/display.

GET /tickets

Retrieve tickets, optionally filtered by submitter or status.

Query Parameters

Parameter Type Description
submitter_user_id string Filter by your app's user ID
submitter_email string Filter by submitter email
status string open, in_progress, pending, resolved, closed
per_page integer Results per page (default: 15)
page integer Page number

PATCH /tickets/{id}

Update the title or content of a ticket.

Request Body

{
  "title": "Updated title",
  "content": "Updated description..."
}
GET /tickets/{id}/comments

Get all public comments on a ticket.

GET /attachments/{id}/view

Render an image attachment inline. This endpoint is only valid for image attachments and requires the same X-API-Key authentication as the rest of the helpdesk API.

Notes

  • Use url to download any attachment.
  • Use view_url only when is_image is true.
  • Non-image files return 422 Unprocessable Entity from this endpoint.
POST /tickets/{id}/comments

Add a comment to a ticket.

Request Body

{
  "content": "I was trying to export January 2026 data."
}

Error Responses

401 Unauthorized — Missing or invalid API key 
{ "message": "Invalid API key" }
404 Not Found — Resource not found 
{ "message": "Ticket not found" }
422 Unprocessable Entity — Validation error 
{
  "message": "The given data was invalid.",
  "errors": {
    "title": ["The title field is required."]
  }
}

Code Examples

PHP (Laravel)

use Illuminate\Support\Facades\Http;

$response = Http::withHeaders([
    'X-API-Key' => config('services.corelink.api_key'),
    'Accept' => 'application/json',
])->post('https://corelink.dev/api/helpdesk/v1/tickets', [
    'title' => 'Export not working',
    'content' => 'Detailed description here...',
    'submitter_name' => auth()->user()->name,
    'submitter_email' => auth()->user()->email,
    'submitter_user_id' => (string) auth()->id(),
    'priority' => 'high',
    'type' => 'bug',
]);

$ticket = $response->json('data');

JavaScript (Fetch)

// JSON request (no attachments)
const response = await fetch('https://corelink.dev/api/helpdesk/v1/tickets', {
  method: 'POST',
  headers: {
    'X-API-Key': process.env.CORELINK_API_KEY,
    'Accept': 'application/json',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    title: 'Export not working',
    content: 'Detailed description here...',
    submitter_name: user.name,
    submitter_email: user.email,
    submitter_user_id: user.id,
    priority: 'high',
    type: 'bug',
  }),
});

const { data: ticket } = await response.json();

JavaScript (with File Attachments)

// FormData request (with attachments)
const formData = new FormData();
formData.append('title', 'Bug with screenshot');
formData.append('content', 'See attached screenshot for the error.');
formData.append('submitter_name', user.name);
formData.append('submitter_email', user.email);
formData.append('priority', 'high');

// Add files from file input
const fileInput = document.getElementById('attachments');
for (const file of fileInput.files) {
  formData.append('attachments[]', file);
}

const response = await fetch('https://corelink.dev/api/helpdesk/v1/tickets', {
  method: 'POST',
  headers: {
    'X-API-Key': process.env.CORELINK_API_KEY,
    'Accept': 'application/json',
    // Note: Don't set Content-Type - browser sets it with boundary
  },
  body: formData,
});

const { data: ticket, attachments } = await response.json();

Best Practices

  • ✓ Store the ticket ID — After creating a ticket, save the returned ID so users can track their tickets.
  • ✓ Use submitter_user_id — Always pass your application's user ID so you can filter tickets by user later.
  • ✓ Include metadata — Use the metadata field to include context like browser info, page URL, or relevant IDs.
  • ✓ Handle errors gracefully — Always check for error responses and display appropriate messages to users.
